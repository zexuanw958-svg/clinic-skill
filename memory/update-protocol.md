# Memory Update Protocol

## Read Before Consultation

Read only memory files relevant to the current case:

- `user-profile.md`
- `recurring-patterns.md`
- `doctor-effectiveness.md`
- `routing-patterns.md`
- recent entries in `consultation-log.md`

If memory is used, say so explicitly.

## Write After Consultation

Never write memory silently.

At the end of a consultation, propose a `可写入病历` section only when there is durable signal:

- a stable user preference
- a repeated problem pattern
- evidence that a doctor was especially useful or unhelpful
- a routing clue that will improve future triage
- a concise consultation-log entry

Only update files after the user explicitly confirms.

## Quality Bar

Do not write:

- secrets
- one-off emotions
- unsupported personality labels
- medical/legal/financial conclusions
- anything the user asked not to remember

Mark memory as `deprecated` if repeated later consultations contradict it.
